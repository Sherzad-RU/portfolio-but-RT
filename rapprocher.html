Compétence : Rapprocher et unifier

Faire converger les technologies réseaux, télécoms, systèmes, développement et cybersécurité.

Rapprocher les technologies dans un projet global

La compétence Rapprocher et unifier consiste à ne pas voir les technologies séparément, mais à les intégrer dans une infrastructure cohérente répondant à un besoin réel.

En deuxième année de B.U.T. Réseaux et Télécommunications, parcours ROM, j’ai travaillé cette compétence à travers la SAE Pépinière, la SAE4 ROM.01, les projets de développement web, les réseaux opérateurs, la ToIP, Docker, PostgreSQL et les ressources de cybersécurité.

Cette compétence m’a permis de comprendre qu’un projet professionnel ne repose pas sur une seule technologie : il faut faire communiquer les réseaux, les serveurs, les services, les applications et la sécurité.

Réseaux Télécoms Systèmes Docker Flask / PostgreSQL Asterisk / ToIP Cybersécurité

Apprentissages Critiques & Preuves de Compétence

AC26.01ROM Intégrer plusieurs technologies dans une infrastructure commune
cliquer pour voir la preuve

Réalisation : Dans la SAE Pépinière, j’ai participé à l’intégration de plusieurs technologies : VLAN, DHCP, DNS, Active Directory, partage de fichiers, site web, téléphonie IP et services Docker.

Méthodologie : J’ai travaillé en reliant progressivement les services entre eux : d’abord le réseau, puis les serveurs, ensuite les services applicatifs et enfin les tests de fonctionnement.

Outils utilisés : Cisco, Linux, Windows Server, Docker, Asterisk, Apache, PostgreSQL, Flask.

Difficultés rencontrées : La principale difficulté était de faire fonctionner plusieurs technologies ensemble sans créer de conflit réseau, de droits ou de configuration.

Ce que j’ai appris : J’ai compris qu’une infrastructure complète demande une vision globale, car chaque service dépend souvent d’un autre.

Preuve : Infrastructure multi-services réalisée dans le cadre de la SAE Pépinière.

Documents associés : 📄 Compte rendu SAE Pépinière
AC26.02ROM Faire communiquer réseaux, services et applications
cliquer pour voir la preuve

Réalisation : Dans mes projets web et réseaux, j’ai relié des applications à des bases de données et des services réseau. Mon projet Flask/PostgreSQL et le portfolio montrent cette capacité à faire fonctionner ensemble développement, serveur et données.

Méthodologie : J’ai commencé par développer les pages, puis j’ai connecté les données avec PostgreSQL, avant de tester l’affichage et les échanges entre client, serveur et base de données.

Outils utilisés : Flask, HTML/CSS, JavaScript, PostgreSQL, SQLAlchemy, GitHub.

Difficultés rencontrées : Les difficultés principales concernaient les routes Flask, les erreurs de connexion à la base de données et la cohérence entre les pages.

Ce que j’ai appris : J’ai appris qu’une application web n’est pas seulement du code : elle dépend aussi de l’infrastructure, de la base de données et de la sécurité.

Preuve : Portfolio web et code source publié sur GitHub.

Preuves associées : 🌐 Voir le portfolio 💻 Code source GitHub
AC26.03ROM Unifier les infrastructures opérateurs et les services clients
cliquer pour voir la preuve

Réalisation : Avec la SAE4 ROM.01 et les TP R3.02, j’ai travaillé sur les réseaux opérateurs et les services clients, notamment avec des notions comme MPLS, BGP, VRF et infrastructures opérateurs.

Méthodologie : J’ai étudié l’architecture globale, puis j’ai compris comment les services clients peuvent être transportés à travers une infrastructure opérateur.

Outils utilisés : GNS3, Cisco IOS, MPLS, BGP, VRF, Wireshark.

Difficultés rencontrées : Les architectures opérateurs sont plus complexes qu’un réseau local, car elles nécessitent de comprendre plusieurs couches et plusieurs protocoles ensemble.

Ce que j’ai appris : J’ai compris que les réseaux opérateurs servent à rapprocher les services, les clients et les infrastructures sur de longues distances.

Preuve : Travaux R3.02 et SAE4 ROM.01 sur les infrastructures opérateurs.

Documents associés : 📄 TP1 R3.02 – Réseaux opérateurs 📄 TP2 R3.02 – MPLS / BGP 📄 SAE4 ROM.01
AC26.04ROM Prendre en compte la sécurité dans l’unification des services
cliquer pour voir la preuve

Réalisation : Dans les projets Cyber et la SAE Pépinière, j’ai compris que l’intégration de plusieurs technologies doit aussi prendre en compte la sécurité : accès SSH, droits utilisateurs, segmentation réseau, GPO, OpenLDAP et bonnes pratiques de cybersécurité.

Méthodologie : J’ai analysé les besoins, puis j’ai appliqué des règles de limitation des accès et de séparation des rôles pour réduire les risques.

Outils utilisés : SSH, ACL, GPO, Active Directory, OpenLDAP, Wireshark, outils Cyber.

Difficultés rencontrées : Il faut sécuriser sans bloquer le fonctionnement normal des services. C’est un équilibre important à trouver.

Ce que j’ai appris : J’ai compris que rapprocher les technologies sans sécurité peut créer des failles. La sécurité doit être pensée dès la conception.

Preuve : Travaux Cyber S3/S4 et SAE Pépinière.

Documents associés : 📄 SAE3 Cyber 📄 SAE4 Cyber.01 📄 Compte rendu SAE Pépinière
