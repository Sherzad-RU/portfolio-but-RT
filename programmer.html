Compétence : Programmer

Créer des outils et applications informatiques pour les réseaux et télécommunications.

La programmation au service des Réseaux et Télécommunications

La compétence Programmer consiste à développer des scripts, des applications et des outils informatiques capables de répondre à un besoin technique dans le domaine des réseaux et télécommunications.

En deuxième année de B.U.T. Réseaux et Télécommunications, j’ai travaillé cette compétence à travers plusieurs projets : création de sites web avec Flask, utilisation de PostgreSQL, développement avec SQLAlchemy, création de pages dynamiques avec Jinja, mise en place d’un panier e-commerce, gestion du multilingue français/anglais et déploiement avec Docker.

Cette compétence m’a permis de comprendre le lien entre une application web, un serveur, une base de données et les échanges client/serveur.

Python Flask HTML / CSS JavaScript PostgreSQL SQLAlchemy Jinja Docker

Apprentissages Critiques & Preuves de Compétence

AC23.01 Automatiser l’administration système avec des scripts
cliquer pour voir la preuve

Réalisation : Dans différents TP Linux et projets, j’ai utilisé des scripts et des commandes pour automatiser certaines tâches d’administration système ou de traitement de fichiers.

Méthodologie : J’ai travaillé en ligne de commande sous Linux pour manipuler des fichiers, vérifier des services, lancer des programmes et organiser mes projets. J’ai aussi utilisé Python pour certains scripts simples, notamment dans des travaux liés aux fichiers réseau ou à l’analyse de données.

Outils utilisés : Linux, Bash, Python, terminal, VS Code, scripts de traitement.

Difficultés rencontrées : Au début, certaines erreurs venaient de mauvais chemins de fichiers, de permissions ou d’une mauvaise compréhension des commandes Linux.

Ce que j’ai appris : J’ai compris que l’automatisation permet de gagner du temps, d’éviter les tâches répétitives et de rendre les manipulations plus fiables.

Ce que je ferais autrement : Je commenterais davantage mes scripts et j’utiliserais une structure plus propre pour faciliter leur réutilisation.

Preuve : L’automatisation de certaines tâches d’administration a été mise en pratique au travers de la SAE Pépinière et de la SAE4 ROM.01, notamment par l’utilisation d’outils Linux, de commandes système et de scripts facilitant le déploiement et l’administration des services.

Documents associés : 📄 Compte rendu SAE Pépinière 📄 SAE4 ROM.01 – Infrastructure opérateur et services clients
AC23.02 Développer une application à partir d’un cahier des charges donné, pour le Web ou les périphériques mobiles
cliquer pour voir la preuve

Réalisation : J’ai développé plusieurs applications web, notamment mon site portfolio et le projet SudoWorks. Ce projet a évolué vers un site dynamique avec plusieurs pages, un affichage de produits, un panier et une gestion multilingue français/anglais.

Méthodologie : Je suis parti d’un cahier des charges fonctionnel : créer un site clair, structuré, avec des pages accessibles et des fonctionnalités précises. J’ai ensuite développé les pages HTML/CSS, ajouté les routes Flask, intégré Jinja pour rendre les pages dynamiques, puis amélioré l’expérience utilisateur.

Outils utilisés : Python, Flask, HTML, CSS, JavaScript, Jinja, VS Code, navigateur web.

Difficultés rencontrées : J’ai rencontré des difficultés sur l’organisation des fichiers, la cohérence du design entre les pages et la gestion du multilingue sans casser la structure existante.

Ce que j’ai appris : J’ai appris à transformer un besoin en application web fonctionnelle, à organiser mes fichiers et à séparer le contenu, le style et la logique serveur.

Ce que je ferais autrement : Je préparerais une maquette plus détaillée dès le début pour éviter de modifier plusieurs fois la structure du site.

Preuve : Développement du site portfolio et du projet SudoWorks avec Flask, HTML/CSS, JavaScript, Jinja, gestion multilingue et structuration des pages selon un besoin défini.

Preuves associées : 🌐 Voir le portfolio 💻 Code source du portfolio
AC23.03 Utiliser un protocole réseau pour programmer une application client/serveur
cliquer pour voir la preuve

Réalisation : Avec Flask, j’ai développé des applications web fonctionnant selon un modèle client/serveur. Le navigateur envoie des requêtes HTTP et le serveur Flask renvoie les pages ou les données demandées.

Méthodologie : J’ai créé différentes routes Flask pour répondre aux requêtes des utilisateurs. Chaque route permet d’afficher une page, traiter un formulaire ou récupérer des données depuis la base.

Outils utilisés : Flask, HTTP, navigateur web, routes Python, templates Jinja, formulaires HTML.

Difficultés rencontrées : Au début, j’avais parfois du mal à comprendre le lien entre une URL, une route Flask, un template HTML et les données envoyées par le serveur.

Ce que j’ai appris : J’ai compris le fonctionnement général d’une application web : le client fait une requête, le serveur traite la demande, puis renvoie une réponse.

Ce que je ferais autrement : Je documenterais mieux chaque route Flask pour expliquer son rôle et les données qu’elle utilise.

Preuve : Développement d’applications fonctionnant selon un modèle client/serveur grâce à Flask, avec mise en œuvre des échanges HTTP entre le navigateur et le serveur. Les compétences en programmation acquises en R3.08 ont également été mobilisées pour structurer le code, gérer les exceptions et manipuler les données.

Preuves associées : 🌐 Tester le portfolio 💻 Code source du portfolio 📄 TP R3.08 – Sérialisation et gestion des exceptions
AC23.04 Installer et administrer un système de gestion de données
cliquer pour voir la preuve

Réalisation : Dans mon projet Flask, j’ai utilisé une base de données PostgreSQL avec SQLAlchemy. J’ai aussi travaillé avec SQLite dans certains essais ou projets plus simples.

Méthodologie : J’ai créé des modèles de données, configuré la connexion entre Flask et PostgreSQL, puis mis en place la création automatique des tables au démarrage de l’application. J’ai utilisé des classes comme User, Project ou Competence selon les besoins du projet.

Outils utilisés : PostgreSQL, SQLite, SQLAlchemy, Flask, Docker, docker-compose.

Difficultés rencontrées : J’ai rencontré des problèmes de connexion entre Flask et PostgreSQL, notamment avec l’URI de connexion, le nom du service Docker, les identifiants et la création des tables.

Ce que j’ai appris : J’ai compris l’importance d’une base de données bien structurée et le rôle d’un ORM comme SQLAlchemy pour manipuler les données depuis Python.

Ce que je ferais autrement : Je définirais plus clairement le modèle de données avant de commencer le développement pour éviter de modifier les tables trop souvent.

Preuve : Mise en œuvre d’une base de données PostgreSQL au sein d’une application Flask, avec utilisation de SQLAlchemy pour la modélisation des données, la création des tables et les opérations de gestion directement depuis l’application.

Preuves associées : 💻 Code source du projet Flask / PostgreSQL 🌐 Tester l’application portfolio 📄 TP R3.09 – Programmation événementielle
AC23.05 Accéder à un ensemble de données depuis une application ou un site web
cliquer pour voir la preuve

Réalisation : J’ai affiché des données issues de la base PostgreSQL directement dans les pages web de mon application Flask. Par exemple, j’ai travaillé sur l’affichage des produits, la gestion du panier et des pages liées aux compétences ou aux projets.

Méthodologie : Les données sont récupérées côté serveur avec Flask et SQLAlchemy, puis envoyées aux templates Jinja pour être affichées dynamiquement dans les pages HTML.

Outils utilisés : Flask, SQLAlchemy, PostgreSQL, Jinja, HTML/CSS, JavaScript.

Difficultés rencontrées : J’ai rencontré des problèmes de doublons dans l’affichage, de synchronisation entre les données de la base et celles affichées sur la page, ainsi que des soucis avec le panier local en JavaScript.

Ce que j’ai appris : J’ai appris à faire le lien entre une base de données, une application serveur et une interface utilisateur.

Ce que je ferais autrement : Je structurerais mieux les fonctions d’accès aux données pour éviter les répétitions dans le code.

Preuve : Accès aux données depuis une application web Flask, récupération des informations stockées en base PostgreSQL avec SQLAlchemy, puis affichage dynamique dans les pages HTML via les templates Jinja.

Preuves associées : 💻 Code source du projet 🌐 Voir l’application web 📄 TP R3.09 – Programmation événementielle

Synthèse personnelle

Cette compétence m’a permis de progresser dans le développement d’applications utiles pour les réseaux et télécommunications. J’ai appris à créer des pages web, à utiliser Flask, à connecter une base de données et à organiser un projet de manière plus professionnelle.

Grâce aux projets réalisés, j’ai compris que la programmation ne sert pas seulement à créer une interface : elle permet aussi d’automatiser, de structurer les données et de faciliter l’administration ou la supervision d’une infrastructure.
